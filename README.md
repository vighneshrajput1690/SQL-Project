# SQL-Project
Loan Credit Risk Management System (SQL)

This project is a SQL-based database management system designed to manage and analyze loan credit data. The system simulates how financial institutions track customers, evaluate credit scores, manage loans, monitor repayments, and assign loan officers.

The project demonstrates practical usage of database design, relational tables, and SQL queries for managing credit risk and loan information.

Key Features

Customer Management
Stores customer details such as name, phone number, email, and city.

Credit Score Tracking
Maintains credit score history for customers to evaluate creditworthiness.

Loan Management
Records loan amount, interest rate, loan date, and loan approval status.

Repayment Tracking
Keeps track of loan repayments and payment dates.

Loan Officer Assignment
Assigns loan officers to specific loans for management and monitoring.

Database Structure

The database contains the following tables:

Customers – Stores customer personal details

Credit_Scores – Maintains credit score records for customers

Loans – Stores loan information and approval status

Repayments – Tracks loan repayment transactions

Loan_Officers – Stores loan officer details

Loan_Assignments – Links loans with assigned loan officers

SQL Concepts Used

This project demonstrates multiple SQL operations including:

DDL (Data Definition Language)
CREATE, ALTER, constraints, primary keys, foreign keys

DML (Data Manipulation Language)
INSERT, UPDATE, DELETE

DQL (Data Query Language)
SELECT, JOIN, GROUP BY, HAVING, ORDER BY, LIMIT, OFFSET

Advanced SQL Concepts

Window functions (RANK())

Aggregate functions (COUNT)

Constraints (CHECK, UNIQUE, NOT NULL)

Table relationships using foreign keys

Objective

The objective of this project is to demonstrate how SQL can be used to design and manage a loan credit risk database, helping financial institutions monitor loans, evaluate customers, and track repayments efficiently.
